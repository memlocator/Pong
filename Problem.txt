Begränsingar:
- Tråd för varje socket
	Inte idealt om man inte kan garantera ett hanterbart antal connections, kan bli väldigt många trådar.
	Kör blockerande sockets eftersom vi kan detta.
	
	Finns OS-funkionalitet för att polla en större grupp sockets åt gången, finns även i biblioteket (SFML).
	Fungerar dock eftersom vi vet att maxantalet connections är lågt.
	
- NAT (network address translation) och portar
	Normalt sett så måste man köra hole punching för att få portarna att fungera mellan klient och klient pga NAT,
	men eftersom klienten redan skickat data till servern via UDP så tas även data mot från servern utan port forwarding.
	
	Skulle vilja att systemet väljer en dynamisk port för att ta emot data på servern från.
	
- Nätverkslagret
	Skulle vilja abstrahera bort sockets mm och implementera ett event subscription system, (observern pattern) 
	där de intresserade objekten notifieras när något med nätverket händer. Kan ju även diskutera hur man kan sortera huruvida eventet är relevant
	eller inte.

- Skriva in IP att connecta tll i spelet

- Välja port för server när man startar den
